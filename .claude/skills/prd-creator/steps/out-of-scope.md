# Out of scope

Based on the locked page inventory, proactively propose a list of likely out-of-scope items — things that *could* be in a data app like this but the user almost certainly doesn't want in v1. Examples common to data apps:

- Write-back / data entry (a data app is read-only against the semantic model)
- What-if parameters and scenario modeling
- Drill-through to row-level detail / exporting underlying data
- Additional pages beyond the locked inventory
- Embedding outside the Fabric portal (a model-connected app currently runs inside the portal only)
- Native mobile layouts, offline mode
- Real-time / streaming refresh, alerting and subscriptions
- Bookmarks, personalization, or per-user saved views
- Multi-language / localization

Present the list, explain why each is a reasonable cut for v1, and ask the user to confirm or pull anything back into scope. Also ask if there's anything else they want explicitly out.
